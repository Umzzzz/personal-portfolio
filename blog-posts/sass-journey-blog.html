Sass - The Learning Journey

View Project

“Sass is the most mature, stable, and powerful professional grade CSS extension language in the world.” (Catlin, Weizenbaum, Eppstein, 2015: online)

Sass stands for Syntactically Awesome Style Sheets. This type of extension is the buzz for front end web developers, as it allows you to write clean, tidy and organised CSS code.

The journey started by doing research around this topic, as it was my first time using Sass. The project was to convert my current CSS into Sass from my major project, Mars Spiders site.

Book Apart collection has a well written book called "Sass for Web Designers” by Dan Cerholm. Reading this book explained the installation process for Sass; I had to install Ruby Gems and Git Bash to help towards the pre-processing and compiling the Sass work flow.

Reading this book gave me a clear understanding of what is required for Sass and what it’s about. Another great resource I used to gain further knowledge was an online tutorial, on Lynda called Sass Essential Training. This help me dig even deeper then the book and gave me further understanding around the coding elements of Sass.

Once the researching was done, it was time to get hands on and implement the Sass into action. The recommended way for the installation of Sass is through Git and Ruby.

I started to install Git and Ruby for my workflow, because of my current coding ability I found it really difficult to use Git to process and compile my Sass. So to overcome this difficulty, I did more research on installation and came across Scout.

“Scout is a cross-platform app that delivers the power of Sass & Compass into the hands of web designers.” (Human, 2016: online)

Scout allowed having an easier work flow, as it help towards pre-processing and compiling the Sass files.

Scout has been launched with my work station.

To set up the Sass work flow, you need to add your work folders, one for the input which is for SCSS file and one for the output which is stylesheet that will show a CSS file once the Sass is complied.

Once you’re done your coding in the back-end with Sass. (Text editor) You need to compile the file. You press play button of the work station and let Scout do its magic by compiling the Sass file.

Once Sass complied through Scout, you can see the CSS file been made. The CSS files shows what line the code is in the SCSS file, whoever reads the files knows exactly what the code is doing.

Variables

Variables were used to store reused code for example font colours, this reduces repetition.

Nesting

Nesting has been used in the navigation code. ul,li and a, selectors are nested in the nav selector. This makes your code organised and readable.

Mixins

Mixins were used in different parts of the coding elements, to group CSS declarations that I wanted to use throughout the site.

Partials

Partials files contain snippet of CSS which can be used in other Sass files. This is a great way to organise your CSS and help keep things easier to maintain.

Extend/Inheritance

Extend is one useful feature when writing Sass. It allows you to share a set of CSS properties from one selector to another.

The journey has come to an end, and it has truly been a success. Exploiting Sass has improved my skills for sure and certainly I will be implementing Sass in projects in the near future, as they ease the coding work flow.
